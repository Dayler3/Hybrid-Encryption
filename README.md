# Hybrid Encryption v1.2

Программный комплекс для защиты данных методом гибридного шифрования. Система использует комбинацию асимметричного алгоритма RSA-2048 и симметричного шифрования AES-256 в режиме CTR. Программа оптимизирована для работы с файлами больших объемов (4 ГБ и более) без перегрузки оперативной памяти.

## Технические характеристики

* **Алгоритм шифрования данных:** AES-256-GCM (блочная потоковая обработка).
* **Алгоритм защиты ключей:** RSA-3072 (OAEP Padding с хешированием SHA-256).
* **Системные уведомления:** Кроссплатформенная реализация через библиотеку Plyer.
* **Интерфейс:** Графическая оболочка на базе CustomTkinter.
* **Оптимизация:** Чтение и запись данных блоками по 4 МБ.

## Системные требования

* Интерпретатор Python версии 3.9 или выше.
* Операционная система: Windows 10/11, macOS 11.0+, дистрибутивы Linux (Ubuntu, Debian, Fedora).

## Установка и настройка

### 1. Установка зависимостей

Для корректной работы всех функций необходимо установить библиотеки из файла конфигурации. Выполните команду в терминале:

```bash
pip install -r requirements.txt
```

### 2. Особенности для Linux

Для работы системных уведомлений в дистрибутивах на базе Debian/Ubuntu требуется пакет `libnotify-bin`:

```bash
sudo apt-get install libnotify-bin
```

## Руководство по эксплуатации

### Начало работы

1. Запустите приложение: `python main.py`.
2. При первом запуске установите мастер-пароль. Он используется для защиты вашего приватного ключа. Потеря пароля сделает расшифровку файлов невозможной.

### Процесс обработки

1. Нажмите "+ ФАЙЛЫ" для добавления объектов в очередь.
2. Выберите режим "ЗАШИФРОВАТЬ" или "РАСШИФРОВОВАТЬ".
3. Во время работы программа отображает текущий прогресс (шкала и проценты) и ETA - расчетное время до завершения операции.
4. По завершении система отправит стандартное уведомление ОС (всплывающее сообщение в углу экрана).

### Остановка процесса

Кнопка "ОТМЕНИТЬ" мгновенно прекращает текущую операцию. Программа автоматически удаляет незавершенный ("битый") файл с диска, чтобы не занимать лишнее место. Ранее успешно обработанные файлы сохраняются.

## Безопасность и хранение ключей

* Ключи хранятся в локальной директории `keys/`.
* Приватный ключ `private_key.pem` зашифрован алгоритмом AES на базе мастер-пароля.
* Программа работает автономно и не использует сетевые соединения.

## Структура проекта

* `main.py` - точка входа в приложение.
* `src/core.py` - криптографическое ядро (RSA/AES).
* `src/gui.py` - графический интерфейс и логика уведомлений.
* `src/processor.py` - управление асинхронными потоками и расчет времени.
* `src/utils.py` - вспомогательные инструменты и работа с файлами.
* `requirements.txt` - список зависимостей (customtkinter, cryptography, plyer).
