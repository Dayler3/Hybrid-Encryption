# Hybrid Encryption

Высокопроизводительный программный комплекс для криптографической защиты данных, реализующий архитектуру гибридного шифрования. Система сочетает стойкость асимметричных алгоритмов с производительностью симметричных шифров, обеспечивая безопасную обработку файлов любого объема.

## Технические характеристики

*   **Симметричное шифрование:** AES-256 в режиме GCM (Galois/Counter Mode). Реализует аутентифицированное шифрование, гарантирующее конфиденциальность и целостность данных (защита от атак типа bit-flipping).
*   **Асимметричный протокол:** RSA-3072 с использованием паддинга OAEP (Optimal Asymmetric Encryption Padding) и хеш-функции SHA-256.
*   **Защита мастер-пароля:** Деривация ключа через алгоритм PBKDF2-HMAC-SHA256 с использованием соли и 100,000 итераций.
*   **Обработка данных:** Потоковая поблочная обработка (размер чанка — 4 МБ), позволяющая обрабатывать массивы данных объемом более 4 ГБ без значительной нагрузки на RAM.
*   **Графический интерфейс:** Оболочка на базе библиотеки CustomTkinter.

### Безопасность и архитектура хранения

*   **Хранение ключей:** Ключевая пара генерируется локально и хранится в директории _keys/_.
*   **Защита Private Key:** Приватный ключ _private_key.pem_ зашифрован на базе мастер-пароля.
*   **Автономность:** Приложение работает полностью локально, не имеет сетевых модулей и не передает данные на внешние серверы.

## Установка
1. Клонирование репозитория

```
git clone https://github.com
```

2. Переход в директорию:

```
cd hybrid-encryption
```

3. Создание и активация виртуального окружения:
>##### — Windows:
>
>```
>python3 -m venv venv
>venv\Scripts\activate
>```

>##### — Linux:
>
>```
>python3 -m venv venv
>source venv/bin/activate
>```

4. Установка зависимостей:

```
pip install -r requirements.txt
```
> _Примечание для Linux: Для работы системных уведомлений может потребоваться установка libnotify-bin_:
>  
>  ```sudo apt install libnotify-bin```

### Зависимости
* Интерпретатор Python версии 3.9 или выше.
* Операционная система: Windows 10/11, macOS 11.0+, дистрибутивы Linux (Ubuntu, Debian, Fedora).

## Руководство пользователя

### Инициализация системы
1. Запустите приложение: `python main.py`
2. При первом запуске необходимо установить мастер-пароль. Он используется для шифрования вашего приватного RSA-ключа.

>_Внимание! В системе не предусмотрено механизмов восстановления пароля. Eго утеря делает дешифровку данных невозможной!_

### Обработка данных
1. Добавьте объекты в очередь через кнопку "+ ФАЙЛЫ".
2. Выберите режим "ЗАШИФРОВАТЬ" или "РАСШИФРОВАТЬ".
3. Интерфейс отображает прогресс выполнения, текущую скорость обработки и расчетное время завершения (ETA).
4. По завершении операции система отправит нативное уведомление ОС.

### Принудительная остановка
Кнопка "ОТМЕНИТЬ" прерывает текущую операцию. Программа реализует механизм автоматической очистки: недописанные ("битые") файлы удаляются с диска немедленно, предотвращая фрагментацию данных.

## Структура проекта
```
├── main.py              - Инициализация и запуск приложения
├── config.json          - Конфигурация и хеш мастер-пароля
├── requirements.txt     - Список зависимостей проекта
├── keys/                - Директория с RSA ключами (PEM)
├── data/                - Директория с выводом файлов(При первом запуске.Её можно изменить)
└── src/
    ├── core.py          - Криптографическое ядро (RSA/AES)
    ├── processor.py     - Движок асинхронной обработки
    ├── gui.py           - Графическая оболочка (CustomTkinter)
    └── utils.py         - Системные утилиты и хеширование
```
